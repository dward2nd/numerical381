Special Cases for IEEE 754 Representation

Zero

It is nearly impossible to represent 000 with IEEE 754. Let's say s=m=e=0s = m = e = 0s=m=e=0, if substituted, then we get:

x=(−1)0⋅(1+0)⋅20=1x = (-1)^0 \cdot (1 + 0) \cdot 2^0 = 1x=(−1)0⋅(1+0)⋅20=1

Regardless of a sign bit, if m=0m = 0m=0 and e=0e = 0e=0, then the value is 000.

Denormalized Numbers

Notice the −c‾- \overline{c}−c in the definition? This biased term was added so that we preserve the maximum value possible (all 1′s1's1′s in the exponent) for the below special case. We call the number without −c‾- \overline{c}−c normalized, and denormalized when e=0e = 0e=0.

If eee is all 1′s1's1′s (i.e., 111111111112=102411111111111_2 = 1024111111111112​=1024; eleven digits of ones) and m=0m = 0m=0, then the value is inf⁡\infinf (infinity).

function mantissa(bitstr::String, debug::Bool = false)::Float64
    bitstr = bitstr[end - 51 : end]
    ret::Float64 = 0.0
    if debug
        @printf("   i | monomial | mantissa \n")
        @printf("-----|----------|----------\n")
    end
    for i in 1:52
        b = parse(Int, bitstr[i])
        tmp = b * 2.0^(-i)
        ret += tmp
        if debug
            @printf(" %3d | %8f | %8f \n", i, tmp, ret)
        end
    end
    
    return ret
end

function exponent(bitstr::String, debug::Bool = false)::Float64
    bitstr = bitstr[2 : 12]
    ret::Float64 = 0.0
    if debug
        @printf("   i | monomial | exponent \n")
        @printf("-----|----------|----------\n")
    end
    for i in 0:10
        b = parse(Int, bitstr[i + 1])
        tmp = b * 2.0^(10 - i)
        ret += tmp
        if debug
            @printf(" %3d | %8f | %8f \n", i, tmp, ret)
        end
    end
    
    ret -= 1023
    if debug
        @printf("==> subtract with 1023: %8f\n", ret)
    end
    
    return ret
end
